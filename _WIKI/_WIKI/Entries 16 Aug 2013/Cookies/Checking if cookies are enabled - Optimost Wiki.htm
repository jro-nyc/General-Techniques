Checking if cookies are enabled

From Optimost Wiki

Jump to: navigation, search

First, drop a session cookie, to ensure that the user has at least 1 cookie set. Then check if document.cookie is an empty string. If document.cookie is empty then the test cookie wasn't set, which means that cookies are disabled or something is wrong with the user's browser.

This is useful if we want to segment out users that have their cookies disabled so that they are not part of the experiment. 

<script>
//Trial code cookie function - drops session cookie "optestcookie"
opCreativeSetCookieA("optestcookie", "1", opCreativeGetDocumentSLD(), 0);

//if document.cookie is empty, test cookie wasn't set, so cookies must be disabled
var cookies = true;
if(!document.cookie || document.cookie == "")
{
	cookies = false;
}

//A trial argument can then be set based on cookies being true or false
var optrial = (typeof(optrial) == "object" && optrial != null) ? optrial : new Object;
optrial.cookies_enabled = (cookies) ? "enabled" : "disabled";
</script>
