Universal Clickthru Code

As we all know, often the most difficult part of creating templates is doing the clickthrus. At the present time, each TC does the clickthrus for their particular Template by adding clickthrus on their own using their own functions. Yes, most are available here in the Wiki, and the TCs and QA folks test to make sure all is working, but the process needs to be standardized as much as possible for our team. This code is going to be used to do just that, but you will still have to do some work to define the clickthrus and tell this code how to "find" them.

Understand that this is NOT an include that you stick on the page and it does ALL the work for you. This is just a way to have every TC use standardized methods to find the clickthrus targets and have the include create the clickthrus.

Methodology of calling the Include

The code is a .js file of about 5k after compression. This is NOT added as javascript code in the template since it would mean every single creative served by Optimost just got 5k heavier, and that would cost us $$$$$. Instead, we put it as an include as the last line of the last module. The code itself will load, define its functions, then, as its last act, call a function - opGenericCT() - that you, the TC, has defined to create the clickthrus. As in: 

if(typeof(opGenericCT)=="function")opGenericCT();

Genesis of the Code

Teddy P gets the credit for this since the root clickthru functions is his creation. The root object is named $o_ct, and all the objects and functions build off that object. This code works for anchor tags, form inputs, and Map/Area tags. A few caveats on the code:

  1. Remember, if the client has js functions that overwrited the action/href onclick, well, it won't work, so remember to look for this.
  2. On a tag with a built-in function that does not use the action/href to redirect - say on onclick or href with a window.open function containing the URL - this will also not work, so look for that too.

Going forward, we may stop tracking these types of redirects.

The code that has been added to Teddy's function is code to finds the "containers" that have the click(s) of interest, and code that takes these container objects and add the clicks. Let's first look at adding the clicks assuming we have the container.

If you have found the "container" with the click(s)

For adding clicks, you have two types of "containers":

  1. A container that may contain many clicks, but you are looking for a single click to add a single clickthru to.
  2. A container with a number of clicks, all of which you want to share the same clickthru.

Individual click

Here is the individual: 

//Object to pass Individual click parameters
individualObject:function(ctVal,elType,startingObj,matchAttribute,matchVal,passThruName,attribs,debugColor){
	this.ctVal=ctVal; //The clikcthru value
	this.elType=elType; //Is it an anchor tag, form tag, or Area tag?
	this.obj=startingObj;  //The starting point to look for this - could be document or some other node
	this.attribute=matchAttribute;  //Attribute we use to identify the link - usually name for form, innerHTML for others 
	this.value=matchVal;  // The value for the attribute we are looking for
	this.qs=passThruName; //The pass thru parameter, we will use, usually oplink or oppost			
	this.attribs=attribs;	//either send an object with a name value pairs to pass,  or null
	this.debugColor=debugColor;  //To "color" the background of the element for testing only.		
},
//Takes elem - which is an instance of individuallObject - and processes/finds the Individual clicks
processIndividual:function(elem){
	if((elem.elType=='a' || elem.elType=='area' || elem.elType=='form') && elem.obj!=null && elem.obj.nodeType==1 
	&& elem.ctVal!='' && elem.qs!='' && elem.attribute!='' && elem.value!=''){
		var elemArray=new Array;
		if(elem.obj.tagName.toLowerCase()==elem.elType) elemArray[0]=elem.obj;		
		else elemArray=elem.obj.getElementsByTagName(elem.elType);				
		for(var i=0;i<elemArray.length;i++){			
			if(elemArray[i][elem.attribute].indexOf(elem.value)!=-1){
				if(elem.elType=='form'){
					if((elemArray[i].method.toLowerCase()=="get" && elem.ctVal.toLowerCase().indexOf('/go/')!=-1)||
					(elemArray[i].method.toLowerCase()=="post" && elem.ctVal.toLowerCase().indexOf('/post/')!=-1)){
						this.setOnclick(elemArray[i], elem.ctVal, elem.qs, elem.attribs);
					}
				}
				else {
					this.setOnclick(elemArray[i], elem.ctVal, elem.qs, elem.attribs); 
				}
			}
		}	
	}
},

So we define the individualObject object, and then call the function processIndividual to process it. Here is what it contains:

  1. ctVal - The clikcthru value
  2. elType - Is it an anchor tag, form tag, or Area tag?
  3. startingObj - The starting point to look for this - could be document or some other node
  4. matchAttribute - Attribute we use to identify the link - usually name for form, innerHTML for others
  5. matchVal - The value for the attribute we are looking for
  6. passThruName - The pass thru parameter, we will use, usually oplink or oppost
  7. attribs - either send an object with name value pairs to pass in the click, or null
  8. debugColor - To "color" the background of the element for testing only.


Containter clicks

Here is the Containter: 

containerObject:function(ctVal,elType,containerObj,passThruName,attribs,debugColor){
//Object to pass Container click parameters
	this.ctVal=ctVal;
	this.elType=elType;
	this.obj=containerObj;
	this.qs=passThruName;
	this.attribs=attribs;	//send an object with name-value pairs to pass, 'count' to tell us to add a number to the URL, or null
	this.debugColor=debugColor;		
},

//Takes elem - which is an instance of containerObject - and processes/finds the Container clicks
processContainers:function(elem){
	if((elem.elType=='a' || elem.elType=='area' || elem.elType=='form') && elem.obj!=null 
	&& elem.obj.nodeType==1 && elem.ctVal!='' && elem.qs!=''){
		var elemArray=new Array;
		var theAttribs=null;
		var countAttribs=null;
		var myNumName=null;
		if(typeof(elem.attribs)=="object") {
			theAttribs=elem.attribs;
			if(elem.attribs.count!=null){
				myNumName=elem.attribs.count;
				theAttribs=new Object;
				countAttribs=new Object;
				theAttribs[myNumName]=0;
				countAttribs[myNumName]=0;				
				for(var n in elem.attribs){
					if(n!='count') theAttribs[n]=elem.attribs[n];
				}
			}
		}
		else if(elem.attribs=='count') {
			myNumName='opNum';
			theAttribs=new Object;
			countAttribs=new Object;
			theAttribs[myNumName]=0;	
			countAttribs[myNumName]=0;				
			for(var n in elem.attribs){
				if(n!='count') theAttribs[n]=elem.attribs[n];
			}
		}
		if(elem.obj.tagName.toLowerCase()==elem.elType) elemArray[0]=elem.obj;		
		else elemArray=elem.obj.getElementsByTagName(elem.elType);
		for(var i=0;i<elemArray.length;i++){
			if(elem.elType=='form'){
				if((elemArray[i].method.toLowerCase()=="get" && elem.ctVal.toLowerCase().indexOf('/go/')!=-1)
				||(elemArray[i].method.toLowerCase()=="post" && elem.ctVal.toLowerCase().indexOf('/post/')!=-1)){
					if(myNumName!=null){
						theAttribs[myNumName]=countAttribs[myNumName];
						countAttribs[myNumName]++;
					}
					else theAttribs=elem.attribs;
					this.setOnclick(elemArray[i], elem.ctVal, elem.qs, theAttribs);
					if(elem.debugColor!=null) elemArray[i].style.backgroundColor=elem.debugColor;
				}
			}
			else {
				if(myNumName!=null){
					theAttribs[myNumName]=countAttribs[myNumName];
					countAttribs[myNumName]++;
				}
				else theAttribs=elem.attribs;
				this.setOnclick(elemArray[i], elem.ctVal, elem.qs, theAttribs);
				if(elem.debugColor!=null) elemArray[i].style.backgroundColor=elem.debugColor;
			}
		}
	}	
},

So we define the containerObject object, and then call the function processContainers to process it. Here is what it contains:

  1. ctVal - The clikcthru value
  2. elType - Is it anchor tags, form tags, or Area tags?
  3. containerObj - The Container we will look in
  4. passThruName - The pass thru parameter, we will use, usually oplink or oppost
  5. attribs - either send an object with name value pairs to pass in the click, or null
  6. debugColor - To "color" the background of the element for testing only.
