Javascript properties nextSibling & previousSibling and problems with them

nextSibling and previousSibling can be useful when stepping through the DOM to access particular nodes or sets of nodes, such as a sequence of table cells. However, they can behave very differently across browsers:

In Firefox, text nodes will often appear between other nodes, whereas they are discounted in Internet Explorer. The following functions use the fact that text nodes have a nodeType property of 3, and can be used in place of nextSibling or previousSibling to get round this: 

function getNextSibling(startBrother){
        if(startBrother.nextSibling){
                var endBrother=startBrother.nextSibling;
                while(endBrother && endBrother.nodeType!=1){
                        endBrother=endBrother.nextSibling;
                }
                return endBrother;
        } else return null;
}

function getPreviousSibling(startBrother){
        if(startBrother.previousSibling){
                var endBrother=startBrother.previousSibling;
                while(endBrother && endBrother.nodeType!=1){
                        endBrother=endBrother.previousSibling;
                }
                return endBrother;
        } else return null;
}
