Functionality

Applies clickthrus onto links or form tags. After reading this, feel free to look at an example of all 3 types of clicks to be tracked: http://kidwww.optimost.com/webtest/jim/testCT.html

When looking at the example, use the debug clickthru tool so you can see what is happening, http://kidwww.optimost.com/webtest/mdauria/qa.html

Instances

For an anchor tag, it should be added as an onclick event, assuming the anchor tag does not already have an onclick, as in: 

//Say the first anchor tag
ckt="{{$clickthru.getClick.url}}";  //How to define the click in Optimost to create a clickthru in the Wizard
ckt="http://by.optimost.com/go/2-3282/a0fb0hw21j245ei27k07n7904/";  //An example clickthru value, an actual value
anchor=document.getElementsByTagName('a');
if(anchor.length>0)a[0].onclick= new Function('$opt.clickthru(this,ckt,"oplink");return true;');

And for Form tag, if it has no onsubmit event already, added as an onsubmit event, as in: 

//Say the first form
ckt="{{$clickthru.getClick.url}}";  
ckt="http://by.optimost.com/go/2-3282/a0fb0hw21j245ei27k07n7904/"; 
document.forms[0].onsubmit= new Function('$opt.clickthru(this,ckt,"oppost");return true;');

or for a Form with an onsubmit already, change the form action straight-up 

//Say the first form
ckt="{{$clickthru.getClick.url}}"; 
ckt="http://by.optimost.com/go/2-3282/a0fb0hw21j245ei27k07n7904/"; 
$opt.clickthru(document.forms[0],ckt,"oppost")

Both these form examples have a "go" clickthru, which assumes the post method="GET". It would be different for a method="POST", and I will talk about that later in this entry.

Code

This first example works for both a anchor tag and a Form POST - note that most of the Forms we deal with have method="POST". And for the absoluteAddr function go to Absolute address. 

$opt={};

$opt.clickthru = function (o, ct, qsParam) {
     if (typeof(o.originaltarget) != 'undefined') return false; //see if we have done this before

     o.originaltarget = 'done'; // Used in tandem with line above to prevent from instrumenting link - doing this process - more than once
     if (o.tagName.toLowerCase() == "form") h=absoluteAddr(o.action);
     else h=absoluteAddr(o.href);   

     o.originaltarget = h;
     h = ct+(ct.indexOf('?')==-1?'?':'&') + qsParam +'=' + escape(h);
     // This gets around problem with '+' characters in URL which are otherwise changed to spaces
     h = h.replace(/\+/g,'%2B'); 
     if (o.tagName.toLowerCase() == "form") o.action= h;
     else o.href= h;

     return true;
}

This next example is more comprehensive as it takes into account whether the Form is a GET or POST, since passing the URL for GET and POST is different - for POST we pass as a query string paramter (just as we would for an anchor tag), but for GET it must be passed as a hidden input. 

$opt={};

$opt.clickthru = function (o, ct, qsParam) {
	if (typeof(o.originaltarget) != 'undefined') return false; //see if we have done this before
		
	o.originaltarget = 'done'; // Used in tandem with line above to prevent from  re-instrumenting the target object
	if (o.tagName.toLowerCase() == "form") h=absoluteAddr(o.action);
	else h=absoluteAddr(o.href);   
		
	o.originaltarget = h;
	newH = ct+(ct.indexOf('?')==-1?'?':'&') + qsParam +'=' + escape(h);
	if (o.tagName.toLowerCase() == "form"){
		if(o.method.toLowerCase()=='post'){//Adding for POST and href is same				
			o.action= newH;
		}
		else {//get Click, need to add Clickthru as action and form post value as a hidden input
			var newInput=document.createElement('input');
			newInput.setAttribute('type','hidden');
			newInput.setAttribute('name',qsParam);
			newInput.setAttribute('value',h);	
			o.appendChild(newInput);
			o.action= ct;			
		}
	}
	else { //Adding for POST and href is same
		o.href= newH;
	}
	return true;
}

Issues

  • Ignores target attribute, function needs be updated to take target attribute into account.
  • Also, remember that a POST click is different than a GET click. Must be defined as POST or GET in the Agency, and it defaults to GET, so creating a POST dynamically means you have to go in and change it from GET to POST in the clickthru. To make this change, go to the clickthru of interest, go to the advanced Tab, and select POST.
  • If we want to add an onclick to an anchor tag, and it already has an onclick, we can do it. First, we need to determine that we can add it to the onclick without adverse effects. If we can, then we can add it within the onclick, usually either at the beginning or the end. Here is an example: 
<a id="testA" href="http://www.optimost.com" onclick="return doSetCookie();">Optimost</a>

<script>

function doSetCookie(){alert('this is a built-in onclick');return true;}

temp=document.getElementById('testA').onclick.toString();  //Get the Anchor object and pull out the onclick event as a string
temp=temp.substring(temp.indexOf("{") + 1, temp.lastIndexOf("}"));  //Removes the function definition tags from the original
document.getElementById('testA').onclick=new Function("$opt.clickthru(this,getClick,'oplink');"+temp); //Add our onclick + orig function

</script>

And here's a function for that: 

function addClickthru(opLinks,clickthru)
{
        if(typeof(opLinks.onclick)!="function") opLinks.onclick=new Function('$opt.clickthru(this,"' + clickthru +'","oplink");return true;');
        else{
                temp=opLinks.onclick.toString();
                temp=temp.substring(temp.indexOf("{") + 1, temp.lastIndexOf("}"));
                opLinks.onclick=new Function('$opt.clickthru(this,"' + clickthru +'","oplink");'+temp);
        }
}


  • Remember that if you add our onclick tracking function at the end, an onclick is expected to return true; in order to allow the href to trigger. So you must remember if you add it at the end to account for that. Also, we add this onclick assuming it ends in return true; since we have changed the href to track it. If the onclick does something else and does not return true, or does something funky to also change the href, or just does something else funky, then adding the onclick may not work and we need to look into more detail in trying to track it.

Example of how to apply with jQuery

Jim's level 2 event function 

$opt.doClickthruJquery = function(evt){
	var randNum=Math.floor(Math.random()*1000000);
	evtX=evt;
	do{  //The event triggered MAY NOT be our anchor tag, so we need to find <a> to get the attributes we are passing in
		if(evtX.is('a')==false)evtX=evtX.parent();
	} while(evtX.is('a')==false && evtX.is('body')==false );	

	var ct="";
	if(evtX.attr('ckthru')!=null)ct=evtX.attr('ckthru');
	var qsParam ="link";
	if(evtX.attr('ckparam')!=null) qsParam =evtX.attr('ckparam');
	h=evtX.attr('href');
	if (h.substr(0,4) != "http"){
		if (h.substr(0,1) == "/"){
		// link is relative to document domain
			h = location.protocol + "//" +location.host + h;
		}
		else { 
		// link is relative to document path
			path=location.pathname.substr(0,location.pathname.lastIndexOf('/')+1);
			h = location.protocol + "//" +location.host + path + h;
		}
	} 
	h = ct+(ct.indexOf('?')==-1?'?':'&') + qsParam +'=' + escape(h);
	evtX.attr('href', h);	
	return true;
}
