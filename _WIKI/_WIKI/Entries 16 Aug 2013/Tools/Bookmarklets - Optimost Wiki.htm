Increment the sticky parameter

If you are QAing creatives using sticky= in the URL, this will increment the sticky parameter by 1.

javascript:increment_sticky:var regexS = "[\\?&]sticky=([^&#]*)";var regex = new RegExp( regexS );var stickyNo = parseInt(regex.exec(window.location.href)[1]);newSticky=stickyNo+1;var newURL=window.location.href.replace("sticky="+stickyNo,"sticky="+newSticky);window.location=newURL;

Decrement the sticky parameter

If you are QAing creatives using sticky= in the URL, this will decrement the sticky parameter by 1.
javascript:decrement_sticky:var regexS = "[\\?&]sticky=([^&#]*)";var regex = new RegExp( regexS );var stickyNo = parseInt(regex.exec(window.location.href)[1]);newSticky=stickyNo-1;var newURL=window.location.href.replace("sticky="+stickyNo,"sticky="+newSticky);window.location=newURL;

Detect Optimost code on the page, "Check for Optimost"

This bookmarlet takes the web page you are on and does a high-level scrub to see if there is any Optimost code on the page.

javascript:var%20e=document.createElement('script');e.setAttribute('language','javascript');e.setAttribute('src','http://asp.optimost.com/webtest/QA/findOpt.js');document.body.appendChild(e);void(0);

If there is no code on the page, it will only detect "itself" on the page, and you will get the alert message saying:

A script call is being made to:
http://asp.optimost.com/webtest/QA/findOpt.js

If there is more on the page, there will be additional information in the alert.

Add Clickthru "value" to the template clickthru page

You need to be at the clickthru page for a template to use this, like https://asp.optimost.com/template-clickthrus.phtml?tid=xyz

javascript:obj=document.getElementById("clickthrus");if(obj){trs=obj.getElementsByTagName("tr");for(var%20i=0;i<trs.length;i++){if((lsv=trs[i].getAttribute("listctrlvalue"))!=null){newTD=document.createElement("td");newTD.innerHTML=lsv;newTD.style.color="#CC0000";trs[i].insertBefore(newTD,trs[i].firstChild);}else%20if(trs[i].parentNode.tagName.toLowerCase()=="thead"){newTD=document.createElement("td");newTD.innerHTML="cVal";newTD.style.paddingRight="0";newTD.className="tdbtngn";trs[i].insertBefore(newTD,trs[i].firstChild);}}}void(0);

Creative and Counter Auto-populater page

We use this for QA, pinning creatives, putting us into the QA Persona, and seeing what counters are available and selected. You must be on the "Vslue Matrix" page at the Wave level, as in:

Adding a 1st party cookie in the domain of a page you are working on

A bookmarklet for dropping a 1st party cookie in the domain of whatever page you invoke this on. It assumes a starting 2-level domain which can be changed manually – so you will change this for international 3-level domains – and will check to see if you are trying to overwrite an existing cookie** or if you are trying to write it to a domain you do not have access to. You can find the bookmarklet here:

http://by.optimost.com/trial/2/p/cookiegenerator-1stparty.5ba/892/content.html

And here is the code:
javascript:var%20e=document.createElement('script');e.setAttribute('language','javascript');e.setAttribute('src','http://es.optimost.com/es/2/c/836/u/addCookie1stParty.js');document.body.appendChild(e);void(0);

You can drag it into the bookmark toolbar in Firefox, Chrome, Safari, for IE just copy in the URL for IE.

So let's say you need to drop a cookie in the acme.com domain, just go to acme.dom, click on the bookmarklet, and a DIV will appear at the top of the page that allow you to drop a cookie. It should be self-explanatory from there.

Uses include:

  1. Setting the Global code selection cookie to point us to QA without adding a QS parameter (For Global implementations that have added 1st party cookies for selection)
  2. For pages with throttling, spoofing the throttling by setting the gum cookie to a value other than "mvt-no".
  3. Changing the value of some existing cookie for some other practical purpose – like hard-coding the SERVERID cookie for the Optimost servers - thus allowing the browser to control which server to invoke - when one of the Pleasenton servers was down.

** Know that this is using javascript to get the cookie information, and when reading the document.cookie object, you cannot determine the exact domain it is in - say acme.com vs www.acme.com - just that you have access to the cookie. This is different than plug-ins/built-in browser functionality that do have access to that information. So when this bookmarklet tells you that the cookie already exists in the domain, it is only looking for a cookie name match, and cannot match the exact domain you entered with the exact domain which it is registered under.
