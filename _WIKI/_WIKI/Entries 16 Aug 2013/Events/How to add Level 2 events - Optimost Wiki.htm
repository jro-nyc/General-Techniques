Adding Level 2 events

Level 2 events are an alternate way of adding events to a node. The main Pros and Cons are:

  • PRO - You can add events without having to worry about other events being overwritten - it just pushes a new event in to also execute for that node.
  • CON - The Level 2 event is associated with the node but is not added to the DOM as a Level 0 event is, so you can't "see it" using a debugger like Firebug.
  • CON - The node that "triggers" the event may not be the node with the event, so using this has to be modified.

Level 2 vs Level 0 issues: Passing Parameters into function and Capture/Bubbling of events

For level 2, there are some differences in how you will have to define the function that is called by the event, and also have to consider the difference between a Level 0 and Level 2 "event".

Function definition

When you do a Level 0 event, you pass the parameters in directly through the event, such as: 

<a id="testAnchor" href="demo.html" onclick="doMyOnclick(this, myValue,"span",true);">Look at Demo</a>

or if I add it to the anchor via jsvascript: 

<a id="testAnchor" href="demo.html">Look at Demo</a>
<script>
obj=document.getElementById("testAnchor");
if(obj){
	if(typeof(obj.onclick)!="function") obj.onclick=new Function('doMyOnclick(this, myValue,"span",true);return false;');
	else{
		temp=obj.onclick.toString();
		temp=temp.substring(temp.indexOf("{") + 1, temp.lastIndexOf("}"));
		obj.onclick=new Function('doMyOnclick(this, myValue,"span",true);return false;'+temp);
	}
}
</script>

For a level 2, you pass the "function" in a similar manner, but do not have to worry about previous existing events. So for a level 2 event, you would say: 

<a id="testAnchor" href="demo.html">Look at Demo</a>
<script>
obj=document.getElementById("testAnchor");
myValue=100;
if(obj){
	$opt.addEvent(obj,"click",new Function('doMyOnclick(this, myValue,"span",true);'),false);
}
</script>

And, as a reminder, the addEvent function looks like this: 

<script>
$opt={};
$opt.addEvent=function(el,evt,func,bubble){
	if(!el || typeof(func) !="function"){return false;}
	bubble = bubble || false;
	var evtType=(window.attachEvent)?"attach":(window.addEventListener)? "add":"none";
	if(evtType == "attach"){el.attachEvent("on"+evt,func);}
	else if(evtType == "add"){el.addEventListener(evt,func,bubble);}
}	
</script>


Passing Parameters vis attributes:CAPTURE/BUBBLING of events

You may also choose to pass parameters via attributes - add attributes to the node with the event, then read them using the "this" property for the node that triggered the event. We read the event object's attributes for click tracking since we need to change the href value to track the click.

You can use "this" for a Level 0 event, but the big difference here is the Capture/Bubbling of Level 2 events . In Level 0, when you get an event triggered, you know 100% that this will be the node with the onclick event. But for Level 2, because of Capture/Bubbling of events, it may in fact be a node INSIDE the node containing the event, so you may have to look through the node triggering the event's parent nodes until you find the node that the event was attached to. So for Level 2, this is not useful.

The Pro and Con of Capture/Bubbling

So two things jump out about Level 2 vs Level 0 as a big limitation and a big advantage.

  1. Limitation - Capture/Bubbling means losing this, which makes passing parameters in via the object's attributes is a little bit trickier, depending on the number of descendant nodes it has. So you have to add code to find the node you attached the event to when the event fires.
  2. Advantage - Capture/Bubbling means you can add an event to a parent container to follow that event inside ALL its descendant nodes. So say you wanted to highlight a node when you mouseover it. You don't have to add an mouseover event to every node - which would take more coding and more resources - you just need to add a mouseover event to the body, and then make changes to the event target that triggered/started the bubbling up of that event.1

Make sure you account for these when using or considering using Level 2.

1Remember that capture/bubbling takes time, so there are limitations. To do click tracking on links that redirect by adding a level 2 event to the body may be dangerous since the page may redirect before it flow up.

How to code around the limitations

There are a number of ways to code around this, a few approaches are listed below:

Salvage the this object

If level 2 events functioned like level 0 events, with respect to the this object referencing the parent of the event handler and not the window, we would not need a work around. As such one solution is to salvage the this object in offending browsers. A popular solution is John Resig's addEvent(): 

<script>
function addEvent( obj, type, fn ) {
  if ( obj.attachEvent ) {
    obj['e'+type+fn] = fn;
    obj[type+fn] = function(){obj['e'+type+fn]( window.event );}
    obj.attachEvent( 'on'+type, obj[type+fn] );
  } else
    obj.addEventListener( type, fn, false );
}
</script>

It differs from the previously discussed addEvent in a few important ways. For IE, it will :

  1. create an additional anonymous function that wraps the passed in function and,
  2. attach the passed in function to the object passed into the function.
  3. so since our function of interest is technically not triggered by the event, but called in the "aftermath" of that event, "this" now has its expected meaning.

It is the second anonymous function that is passed to the attachEvent() function. It is the "true" event handler but will act as a reference to the original function, and because the original function is attached to the original object and it is passed the event object, it can be treated exactly like an event handler that correctly uses this.

One gotcha to be aware of: this version assumes that type+fn is unique on obj. This could potentially lead to name collisions.

Loop to parent

Alternatively we could use the event object to find the correct element:

So let's take a clickthru function we've used before and show how we get around some of the Level 2 event limitations.

  1. "evt" is the event - which can be passed into a Level 2 event - and to find the node that triggered the event, we look for evt["srcElement"] in IE or evt["target"] in the other browsers. Once found, I am calling the node evtX.
  2. With evtX in hand, in order to find the node of interest - the "this" workaround --- in this example we will assume that we are looking for an anchor tag that contains an image, so since the event may trigger from the image, if our node is not the anchor tag, we need to look at its parentNode to find the anchor tag. In case there are more nested elements, it is a do{...}while loop that continues until it finds an anchor tag OR reaches the body tag.
  3. You will notice that I am looking for an attribute of the node that contains the clickthru value - evtX.ckthru - and an attribute that contains the name of my query string paramter - evtX.ckparam. As far as I have seen, you cannot pass parameters into Level 2 events, so I pass them in by attaching them as attributes for that node (not here, but I would do this when I define the Level 2 event to call this function). 
<script>
$opt.doClickthru = function(evt){
	var randNum=Math.floor(Math.random()*1000000);
	evtX=(evt["srcElement"]?evt["srcElement"]:evt["target"]);
	
	do{  //The event triggered MAY NOT be our anchor tag, so we need to find <a> to get the attributes we are passing in
		if(evtX.tagName.toLowerCase()!="a")evtX=evtX.parentNode;
	} while(evtX.tagName.toLowerCase()!="a" && evtX.tagName.toLowerCase()!="body" );	

	if (typeof(evtX.originaltarget) != 'undefined') return false; //see if we have done this click before
	evtX.originaltarget = 'done'; //mark with click so we won't do it again

	var ct="";
	if(evtX["ckthru"]!=null)ct=evtX["ckthru"];

	var qsParam ="link";
	if(evtX["ckparam"]!=null)qsParam =evtX["ckparam"];
	if (evtX.tagName.toLowerCase() == "form") h=evtX.action;
	else h=evtX.href;
	if (h.substr(0,4) != "http"){
		if (h.substr(0,1) == "/"){
		// link is relative to document domain
			h = location.protocol + "//" +location.host + h;
		}
		else {
		// link is relative to document path
			path=location.pathname.substr(0,location.pathname.lastIndexOf('/')+1);
			h = location.protocol + "//" +location.host + path + h;
		}
	} 
	h = ct+(ct.indexOf('?')==-1?'?':'&') + qsParam +'=' + escape(h);
	//alert(h);
	if (evtX.tagName.toLowerCase() == "form")
		evtX.action= h;
	else
		evtX.href= h;
	return true;
}
</script>

And we create this event from an anchor tag we want to track: 

<script>
var opPhoneButton = document.getElementById("insuranceLink");
if(opPhoneButton ){
	opPhoneButton.ckthru="{{$creative.sessionId}}";
	opPhoneButton.ckparam="link";
	$opt.addEvent(opPhoneButton,"click",$opt.doClickthru,false);
}
</script>
