Overview

In order to make changes to an iFrame from a parent document, it must be in the same domain as that parent document. Assuming that is true, here is an simple example of that would be done.

Scenarios where you cannot access the iframe

The rule is that one can access the content inside an iframe if the iframe is in the same domain as the parent container. That means the following:

  1. It has to be the exact same domain. So:
    • www.acme.com and www.acme2.com are obviously not the same domain.
    • www.acme.com and shop.acme.com are also not the same domain for iframe access.
  2. If the iframe uses the https: protocol, then the parent frame must also use the https: protocol.
  3. If the domains and protocols match up, but the iframe src location does a 302 redirect, then the redirect's domain is the domain that must match. If you have issues with accessing what appears an iframe in the correct domain, check for this using the .NET tab in Firebug or the appropriate tool in Chrome or IE.

Adding/changing content inside an iFrame

Our simple example can be found at the URL:

http://asp.optimost.com/webtest/jim/iFrames/index.html

This file's only content is iFrame inside the body tag, which is followed by script code for the manipulation. The URL of the iFrame is:

http://asp.optimost.com/webtest/jim/iFrames/car_unlogin.html

Here is the content of the host file, with comments where appropriate: 

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">
<html>
<head>
	<title>iFrame modification, when iFrame is in same domain</title>
</head>
<body>
<IFRAME id="car" marginwidth="0px" marginheight="0" src="car_unlogin.html" width="1054px" height="305px" frameborder="0" scrolling="no"></IFRAME>
<script>
myIframe=null;
iframes=document.getElementsByTagName('iframe');
for(var i=0;i<iframes.length;i++){
	if(iframes[i].id=='car'){
		iframes[i].onload=new Function('changeFrame();');
		myIframe=iframes[i];
		break;
	}
}
function changeFrame(){
	if(myIframe!=null){
		alert('in');
		var iDocument = (myIframe.contentWindow || myIframe.contentDocument);
		if (iDocument.document) iDocument = iDocument.document;
		if(iDocument!=null){
			var opAnchor = iDocument.body.getElementsByTagName('a');			
			for (var i=0;i<opAnchor.length;i++) {
				if(opAnchor[i].href.indexOf('carInsuranceApply') != -1) {
					if(opAnchor[i].parentNode.className=='li_r3') {
						//The code below takes the Unicode representation of the KANJI characters and adds it to a string 
						//using the javascript "fromCharChode" method.  Since String is a protected word for a string,
						//String.fromCharCode(abc);	returns the character representation for the UNICODE values
						kj=[31435,21051,33719,24471,25253,20215];
						str='';
						for(var k=0;k<kj.length;k++){
							str+=String.fromCharCode(kj[k]);	
						}

						//We are going to replace the contents of the anchor tags, so lets remove all its children first
						var children=opAnchor[i].childNodes;
						for(var k=children.length-1;k>-1;k--){
							children[k].parentNode.removeChild(children[k]);
						}
										
						// Now we create the new element to go inside the anchor tag.																				
						// You have to create an element in the RIGHT DOCUMENT.  So you can'd say document.createElement, to
						// iDocument.createElement, where iDocument is the document object for the iFrame - 
						// iFrameObj.contentWindow.document or iFrameObj.contentDocument.document

						//This code creates a new span tag element, added to the iFrame's document node, iDocument.
						var newSpan=iDocument.createElement('span');
						newSpan.innerHTML=str;
						opAnchor[i].appendChild(newSpan);		
						
						//Here is code for adding it as a text node															
						//newTextNode=iDocument.createTextNode(str);						
						//opAnchor[i].appendChild(newTextNode);											
					}
				}
			}
		}	
	}
}
</script>		
</body>
</html>

Explanation of how we do it

The goal of this exercise is to find a particular anchor tag in the iframe - the one that contains carInsuranceApply in its href and has a parentNode with class="li_r3" - and replace its content with something new. Notice this is a Chinese language page - just to make it interesting, feel free to read the Dealing with Kanji entry for more background - but the steps only get modified slightly because of this.

So in order to do this, we do the following:

  1. Immediately after the iFrame is defined, we add an onload event that will execute our function once the iFrame is done loading. I find all the iFrames on the page, then look for the one with id="car". Ok, I realize I did not need to do this since there is only one iFrame, but makes sure you add the onload event to the right iFrame.
  2. I set the global variable myIframe equal to the iFrame object
  3. When the iFrame loads and changeFrame() is called, the function finds the node of interest, and makes the appropriate changes. This is a very standard DOM manipulation, but with a few things to note:
    1. The key is finding the "document" object for the iFrame in order to modify its DOM tree. I found code for how to do this, and since it is different browser to browser, there is a decision statement that determines that. I call the iFrame's document iDocument. This makes sense since we are doing all this DOM manipulation from the parent document. If we made changes from inside the iFrame, then we would use document for the iFrame and then find the parent document object and name it something else to make DOM changes to the parent.
    2. Since we have a different document object for the iFrame - which means all references to document refer to the parent from which we make those changes - all DOM operations need to use this as the iFrame's "document". So document.getElementById() is now iDocument.getElementById(), document.createElement() is now iDocument.createElement(), etc. I discovered this when I could not use appendChild on a new node I had created, but could modify the innerHTML on an existing node.
