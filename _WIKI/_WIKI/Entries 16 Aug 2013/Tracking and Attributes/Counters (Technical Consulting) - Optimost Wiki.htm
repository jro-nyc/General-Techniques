What is a Counter?

A counter is a "web beacon" that allows you to record visitor events that take place after an impression. For example, placing a counter on the last page of a subscription process will allow you to view the "conversion rate" for each creative. Each event is associated with the original impression creative.

Counters are associated with a particular campaign. Choose the Counters task from any Campaign Details page to manage counters for that particular campaign.

A single counter can record activity for multiple subjects within a campaign. For example, if you are testing two different subjects that lead to the same sales funnel, you only need a single counter to track activity for both subjects. The activity tracking topic describes the relationship between subjects and counters in-depth.

Viewing Counters List

Counters are associated with a particular campaign. To view and manage counters for a campaign, go to the Campaign Details page for a campaign and choose the Counters task. The Counters list displays all counters related to a campaign.

  • Click New to create a new counter.
  • Select a counter and click Edit to change details for that counter.
  • Select a counter and click Delete to delete that counter.
  • Select a counter and click Get Code to get the code for that counter.

Creating and Editing Counters

Counters are managed from the Campaign Counters List page. New counters can be created by clicking the New button under the Counters list. To change counter details for an existing counter select a counter in the Counters list and click Edit. Note: You can create a counter if you are the administrator or if your user account has the appropriate permissions.

Counter Fields

Each counter is configured through the Counter Details dialog.

General Tab

General counter properties.

FieldDescription
NameName should identify the counter. This name will be used as the default column name in reports.
DescriptionNotes or comments about the counter. (Optional)
Serve AsDetermines how the counter is invoked or triggered. The two invoke methods are JavaScript and binary image.

It is recommended that counters are invoked using JavaScript. JavaScript counters are invoked using a SCRIPT block and therefore have access to the browser environment and can automatically pass counter-related information to the counter.

Binary images are invoked as an IMG tag. Because browsers can render images asynchronously from the page metrics may not be as accurate as when using JavaScript counters. Also, images do not have access to the browser environment which makes you responsible for appending any counter-related information to the counter query string.

If the counter will be invoked from a secure page you should check Counter is activated from a secure page. This option will make sure all URLs for the counter are secure (https:). If you fail to check this box when the page is secure the browser may prompt visitors and tell them insecure content is being referenced from a secure page.

Subjects Tab

All subjects in the counter campaign are shown in the subject list. Click the checkbox next to subjects that the counter should track activity for. If a subject is not checked, the counter will not record activity for that subject. You can view details for a subject by selecting a subject and clicking Details. Subject details can only be viewed however, not modified from this dialog.


The list shows the name of each subject and the query string argument the counter will look at for impression information about that subject. Counters invoked using JavaScript will include this query string argument automatically if you are using cookies for activity tracking.


Values Tab

Counter values allow you to assign a decimal value to each action. You can record two different values for each action.

  • No value
    Select this option if no value is associated with actions. This is the default.
  • Value is always
    If the value for each action is constant you can specify the value with this option.
  • Value is specified by
    If the value for each action is dynamic you can choose the value when invoking the counter.
    Counters invoked using JavaScript can look for the action value from four sources: a browser query string argument, the value of a browser cookie, the value of a JavaScript variable or by executing a JavaScript expression.
    Depending on the option you select you need to set the value source to be the name of a query string argument, the name of a cookie, the name of a JavaScript variable or a valid JavaScript expression (such as a function call).
    You can specify a default value that will be used in the case when the value source does not contain a value. If you do not specify a default value no value will be associated with the action unless the value source provides a valid value.
    The request argument is the name of the counter query string argument that will contain the value. If you don't provide a name one will be created based on the value source.

Attributes Tab

Counter attributes allow you to capture up to two string values and associate them with each action.

FieldDescription
NameThe name of the attribute. This name will be used as the default column name in reports.
Value is specified byCounters invoked using JavaScript can capture the attribute value from four sources: a browser query string argument, the value of a browser cookie, the value of a JavaScript variable or by executing a JavaScript expression. Set value is specified by to the appropriate source.
Attribute sourceDepending on the option you select you need to set the attribute source to be the name of a query string argument, the name of a cookie, the name of a JavaScript variable or a valid JavaScript expression (such as a function call).
Default valueYou can specify a default value that will be used in the case when the source does not contain an attribute value. If you do not specify a default value no attribute will be associated with the action unless the source provides a valid value. (Optional)
Request argumentThe name of the counter query string argument that will contain the attribute value. If you don't provide a name one will be created based on the value source. (Optional)

Deleting Counters

Counters are managed from the Campaign Counters List page. To delete a counter, select the counter in the Counters list and click Delete.

When you delete a counter, metrics for the counter will still be included in report aggregate counts but custom columns for the counter will be removed. No new subject activity will be recorded after the counter is deleted.

Note: You can delete a counter if you are the administrator or if your user account has the appropriate permissions.

Getting Counter Code

To get code for a counter, go to the Campaign Details page for the campaign associated with the counter you want code for. Choose the Counters task then select the counter in the Counters list and click Get Code.

Counter Attributes and Values

You can capture query string values from counter requests and include them in reports.


To change attributes and values for an existing counter, change counter details for the counter you want to edit.


Attribute values can be any sequence of characters up to 32 characters in length. You can customize reports to break down aggregate metrics by attribute values.


Decimal values can be in the range -9,999,999.99 to 99,999,999.99. Only the first two digits following the decimal point are recorded. You can customize reports to show value totals and averages.
