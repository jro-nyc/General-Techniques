Session Identifier

Here is some information on the session identifier

Format

You have seen our Session Identifiers as cookie values, dropped as part of the javascript returned in a content.js call to the Optimost Servers. Here is an example Session Identifier you might see returned as a query string parameter in the content.js call to our servers, or dropped as a first-party cookie.

op24runruntestliid=a0ru26teqa283dl04m37ye22d

Details

We can say the following about the Session Identifier:

  • The value for each field is an integer converted to BASE-36.
  • This means that each 3-digit executor can serve NO MORE THAN 363-1 or 46,655 requests and an individual "Server ID" can serve NO MORE THAN (46,655 * 46,655) requests in a day.
  • Individual identifiers are created by concatenating the generated space prefix with the request index and checksum.

The content

Here is what each digit contains.
// Creates a session identifier based on the request context.
// Session will be 25 or 37 characters in length.
//
// z Format version
// zzz SegmentID (46,655 max)
// zzz WaveID (46,655 max)
// zzz CreativeID (46,655 max)
// zzzzzzzzzzzz VisitorID
// zzzzzzzzzzzz ImpressionID (not included if same as VisitorID)
// xxx Checksum (partial MD5)

The ImpressionID/VisitorID

Auto-generated IDs are a 12-character string divided into 8 fields:


// BIT FIELD RANGE
// z Version 1-35
// z Year 1-35
// z Month 1-12
// z Day 1-31
// z Server ID 1-35
// zzz Executor 1-46,655
// zzz Request count 1-46,655
//  ? Checksum [0-9A-Z]

Our Session Identifier

Let's use this information to break down our Session Identifier a0ru26teqa283dl04m37ye22d, created on March 3, 2012


a                       Format version, version 10
0ru                    SegmentID (46,655 max) 1002
26t                     WaveID (46,655 max) 2837
eqa                   CreativeID (46,655 max) 19090
283dl04m37ye   VisitorID, broken down as follows:
                         version=2
                         year=8
                         month=3
                         day=13
                         serverID=21
                         Executor=166
                         request Count=4154
                         Checksum=e (decimal equivalent not relevant for this parameter)
____________    ImpressionID (not included if same as VisitorID, and not included here)
22d                   Checksum (partial MD5)
