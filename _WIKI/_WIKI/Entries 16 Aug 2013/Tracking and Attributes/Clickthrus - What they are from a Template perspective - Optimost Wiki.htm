What is a Clickthru?

A clickthru is a special tracking link that is used to monitor clicks on links in a creative. Clickthrus are created by the Template Wizard when the template is created. The wizard replaces links that are being tracked with references to special clickthru variables. Clickthru variables have the notation {{$clickthru.linkname.url}}.

A "GO" Click

First type of tracking is on a GO click - this would be hrefs in anchor tags/imags maps, or a Form with method="GET". If it is a GO click, you will see the words "go" in the URL, as in:

goClick="http://by.optimost.com/go/1-32/a0fb0h1j245ei27n7904/";

A you can see, there is now an intermediate step where the browser goes first to our servers, and is then sent to the actual URL of interest. A few notes on this process:

  1. This is standard practice on the Web, not considered "underhanded" in any way. This is what Google, Yahoo, and all other advertisers do to track users clicking on a search result. Let any wary customers understand this.
  2. This transaction is purely between the visitors browsers and the Optimost servers and the Customer's servers. If they are passing "sensitive data", when it comes to Optimost, it is just sent back to the visitor's browser with the correct, final URL. A secure connection will also be secure to our servers. And we do not store all the data from the click, we just count the click, only store any attributes and values explicitly defined to be collected for that click, and then redirect it. The only new vulnerability introduced is that the Optimost servers are now involved. They should trust us - out company's reputation depends on it - but if their security folks say "NO!", we can simply choose to not track the click, which means we will just have less data for analyzing their page, but can put a counter on the next page to measure visitor's successfully filling out the form.

A "POST" Click

The other type of tracking is on a POST click - this would be a Form with method="POST". If it is a POST click, you will see the words "post" in the URL, as in:

postClick="http://by.optimost.com/post/1-31/a0fh1j263deei27n7904/";

In the case when we are tracking a Form with method="POST", we cannot use a 302 redirect. A 302 assumes that all the pertinent information is in the URL, which is the case for anchor tags and Forms with method="GET" since any extra information is passed as query string parameters in the URL. But for a POST, the data of interest is passed in the HTTP headers, so we must use a different technique.

The technique works like this: the visitor POSTs the data, which since we are tracking, comes back to Optimost. Now, since the click says "post", Optimost now takes the posted name-value pairs in the HTTP headers, the proper destination post URL, and constructs and sends an "intermediate" HTML page with no visible content back to the visitor's browser. That page has a form with all the name-value pair information as hidden inputs with the corresponding names and values. This "intermediate" page's body tag will have an onload event so that as soon as it gets to the visitor's browser and loads, it will post all the pertinent information to the final URL. So the visitor submits the form, which comes back to Optimost. Then, after tracking the click, Optimost will send back to the browser the following HTML: 

<HTML>
<HEAD><TITLE>Untitled</TITLE></HEAD>
<BODY onload="f[0].submit();">
<FORM style="margin: 0;" action="http://www.optimost.com/" method="post">
<INPUT TYPE="hidden" NAME="CanineName" VALUE="Bogie">
<INPUT TYPE="hidden" NAME="CanineGender" VALUE="Female">
<INPUT TYPE="hidden" NAME="CanineBreed" VALUE="Beagle">
<INPUT TYPE="hidden" NAME="TYPE_CONST" VALUE="Leg sniffer">
<INPUT TYPE="hidden" NAME="city" VALUE="Laurel, MD">
<INPUT TYPE="hidden" NAME="food" VALUE="Dinner table scraps">
<INPUT TYPE="hidden" NAME="likePeopleBecause" VALUE="They rub my belly">
<INPUT TYPE="hidden" NAME="x" VALUE="72">
<INPUT TYPE="hidden" NAME="y" VALUE="10">
</FORM>
<SCRIPT LANGUAGE="JavaScript">
var f = (typeof document.forms == "object") ? document.forms : document.getElementsByTagName("form");
</SCRIPT>
</BODY>
</HTML>

You can test this yourself - put it in Homesite or save as HTML and open in a browser, and when you view it, you will see it redirect to optimost.com.


A few notes on this process:

  1. If the visitor hits the back button, it goes back, not to the form page, but the "intermediate" page (clickthru page), the blank page that has the form the submits onload. Again, no way to avoid this page unless we decide not to track the POST click.
  2. Since the POST data going to the client's servers is now coming from the "intermediate" page, if the client is tracking document.referrer on this POST, it will be coming from Optimost since that "intermediate" page came from Optimost.
  3. If the Form has an input with name="submit", then the onload event cannot use the submit method, and we can't track the click. OUCH... This is actually more common than one would think, since if you name a submit button "submit" this will happen. The GECKO DOM reference warns coders not to do this, here is the link to the form.submit():

http://developer.mozilla.org/en/docs/DOM:form.submit

Viewing Clickthrus List

Clickthrus can be viewed and managed from either the Subject Clickthrus List page or the Template Clickthrus List page.

  • To view all clickthrus for a subject, go to the Subject Details page for a subject and choose the Clickthrus task.
  • To view all clickthrus for a template, go to the Template Details page for a template and choose the Clickthrus task.

Creating Clickthrus

Clickthrus can only be created by the Template Wizard. If the template contains HTML content, a clickthru can automatically be created for the following elements:

ElementAttribute
A (anchor)href
AREAhref
FORMaction

Links in both the template copy and variable values can be converted into clickthrus. Only valid links to Web pages will be converted. Links containing JavaScript script or internal references (#) are ignored.

The wizard creates clickthrus with default properties. You can customize clickthrus by editing the clickthru after the template is created.

You can explicitly create clickthrus for links the the wizard cannot find. For example, if you want to track clicks on a URL referenced in JavaScript, you can replace the URL with a clickthru variable before running the wizard. To explicitly create a clickthru, add a reference to a variable named {{$clickthru.linkname.url}} where linkname is the name of the clickthru. The wizard will see the clickthru and assign it an empty URL. You can then edit the clickthru and set the target URL after the template is created.

To add clickthrus to an existing template, edit the template using the Template Wizard. Clickthrus will be created for references to new links in the template.

Clickthru variables must be referenced in the template copy or variable values, they cannot be referenced in the preview shell. After changing template clickthrus, you must refresh creatives in order for creatives in active waves to reflect the changes.

Editing Clickthrus

Clickthrus can be managed from either the Subject Clickthrus List page or the Template Clickthrus List page. To change details for an existing clickthru, select a clickthru in the Clickthrus list and click Edit. After changing template clickthrus, you must refresh creatives in order for creatives in active waves to reflect the changes.

Clickthru Fields

Each clickthru is configured through the Clickthru Details dialog.

URL Tab

Each clickthru must have a target URL defining where visitors will be redirected after the click is recorded. The template wizard will set this target for clickthrus it creates for HTML elements. If you explicitly create clickthrus you must make sure to set the target yourself. If the target is dynamic you can configure the clickthru so that the target can be specified or overridden at runtime by a clickthru query string argument.

  • Redirect visitor to
    Visitors will be redirected to this target. Check URL can be overridden by request argument if the target can be overridden by a query string request argument. Specify the request argument name in the input field. If the request argument is not in the query string or does not contain a valid URL, the default target will be used.
  • URL will be always specified by request argument
    Choose this option if the target is always dynamic and there is no default target. Specify the request argument name in the input field. If the request argument is not in the query string or does not contain a valid URL, the visitor will be shown a browser defined error page.


Typically you need to configure a clickthru passthru for each clickthru for which the target is specified or overridden by a request argument. Targets must be an absolute URL. For example, Web page targets must begin with http:// or https://.

Click Value Tabs

Clickthru values allow you to assign a decimal value to each click. You can record two different values for each click.

  • No value
    Select this option if no value is associated with clicks. This is the default.
  • Value is always
    If the value for each click is constant you can specify the value with this option.
  • Value is specified by request argument
    If the value for each click is dynamic, you can specify that the value will be made available through a clickthru query string argument. The request argument is the name of the clickthru query string argument that will contain the value. Typically you also need to configure a clickthru passthru for the clickthru.
    You can specify a default value that will be used in the case when the request argument does not contain a value. If you do not specify a default value no value will be associated with the click unless the value source provides a valid value.

Attributes Tab

Clickthru attributes allow you to capture up to two string values and associate them with each click.

FieldDescription
NameThe name of the attribute. This name will be used as the default column name in reports.
Request argumentThe name of the clickthru query string argument that will contain the attribute value. If you don't provide a name one will be created based on the value source. Typically you also need to configure a clickthru passthru for each clickthru for which an attribute is captured. (Optional)

Advanced Tab

FieldDescription
Inherit settings from clickthru groupsIf this option is checked, clickthru values and attributes will be overridden by the settings for any clickthru group the clickthru is a member of.

Deleting Clickthrus

Clickthrus can only be deleted by the Template Wizard.

To remove clickthrus from an existing template, edit the template using the Template Wizard and remove references to the clickthru from the template copy and variable values.

After changing template clickthrus, you must refresh creatives in order for creatives in active waves to reflect the changes.

Getting Clickthru Code

Important: Clickthru code is provided for diagnostic purposes only. You should not normally need to use the code directly. Clickthru code is not intended to be referenced externally. Clickthru variables are automatically replaced with the appropriate code when creatives are generated.

Clickthrus can be managed from either the Subject Clickthrus List page or the Template Clickthrus List page. To get code for an existing clickthru, select a clickthru in the Clickthrus list and click Get Code.

Clickthru Attributes and Values

You can capture query string values from clickthru requests and include them in reports.

To change attributes and values for an existing clickthru, change clickthru details for the clickthru you want to edit.

Attribute values can be any sequence of characters up to 32 characters in length. You can customize reports to break down aggregate metrics by attribute values.

Decimal values can be in the range -9,999,999.99 to 99,999,999.99. Only the first two digits following the decimal point are recorded. You can customize reports to show value totals and averages.

Typically you need to configure a clickthru passthru for each clickthru that is capturing values.

Viewing Clickthru Groups List

Clickthru Groups are managed from the Subject Clickthru Groups page.

To view and manage Clickthru Groups for a subject, go to the Subject Clickthrus List page for a subject and choose the Clickthru groups task.

The Clickthru Groups list displays all clickthru groups for the subject. Clickthrus for each group are grouped together by template.

  • Click New Group to create a new group.
  • Select a group and click Edit to change details for that group.
  • Select a group and click Delete to delete that group.
  • Click on a template name to go to the Template Details page for that template.
  • Click on a clickthru target to open a window displaying that target.
