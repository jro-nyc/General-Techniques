Creating an Image Counter

From Optimost Wiki

Jump to: navigation, search

Why would you need to do this?

Well there are a number of reasons

  • Want to count some state when a button is clicked, and view as counters are viewed in Wave reports - this is in fact the exact context in which this one was created.
  • Can't track this click but want stats - client may not want us to do a 302 redirect for some reason, but still want info.
  • Can't insert counters in flow - valid reason, of course you could use a click to track it, but whatever.

In the example below, they wanted to know which of the two radio buttons was selected when the visitor clicked on either the "click here to join" button or the "Try us free for 30 days" link.

You can take the code and view source, you will see that the function doOptInCheck() pulls the value of the selected radio button and sets a tag accordingly. The tag already contains the URL of the image counter, and the creative ID. We then add these attributes (host and testrun) to give us full reporting info.

Right now, we have are triggering an alert with the info. Notice that the {{$creative.sessionId}} is appearing as escaped content, in the template it would be given a session identifier. And for the real application, comment out the alert tags and uncomment the img.src = tag; statements.

Code

<script language="javascript">
function doSubmitFunction(a,b,c){document.theForm.submit();}

var tag = 'http://by.optimost.com/counter/2/-/2/event.gif?op2testaliid=' + escape("{{$creative.sessionId}}") + '&';
var img = new Image;
function doOptInCheck(val,btn){
	if(btn=='signup') join=true;
	else join=false;	
	if(val==12) {
		if(join) tag += 'host=securityPlus&testrun=signup';
		else tag += 'host=securityPlus&testrun=free';
		alert(tag);
		//img.src = tag;
	}
	else if(val==1) {		
		if(join) tag += 'host=unlimited&testrun=signup';
		else tag += 'host=unlimited&testrun=free';
		alert(tag);
		//img.src = tag;
	}
}
var notPosting = true;
</script>

Caveats

There are a few consideration when doing this.

  1. Not reliable if page redirects. This use to work back in the day, no longer. Don't use it for redirecting page, period. Only use for new pages being spawned or events that trigger non-redirecting functionality - like hidden div areas showing, Ajax calls, etc.
    UPDATE July 2010 - We are finding them to be more reliable on redirects, >95%, to the point that we are using them more when there are issue with clickthrus. Why the improvement? Might be a newer browser thing, not sure.
  2. If you are clicking multiple times, remember that images get cached and you might only see it the first time. If you are just sending the session ID and some attribute, resending it may not retrigger the image. To get around this, you can just add a query string paramter with a random number attached.
  3. Make sure the image has global scope. The image may not immediately trigger, so if it is defined inside a function, when the function closed, it is garbage collected out, which may happen before it triggers. So if you define a global array and make the image part of that array, then you will be fine. 
var myImgs=new Array;
doBinaryCounter = function(_val){
	var _tag = "http://by.optimost.com/counter/0/-/0/event.gif";
	myImgs[myImgs.length] = new Image();
	myImgs[myImgs.length-1].src = _tag+ "?op0testMVTliid=" + escape("{{$creative.sessionId}}") + "&opAttribute=" + _val;
}

Second time may not work, so instead have a random number tha changes the URL. 

var myImgs=new Array;
doBinaryCounter = function(_val){
	var randNum=Math.floor(Math.random()*1000000)
	var _tag = "http://by.optimost.com/counter/0/-/0/event.gif";
	myImgs[myImgs.length] = new Image();
	myImgs[myImgs.length-1].src = _tag+ "?op0testMVTliid=" + escape("{{$creative.sessionId}}") + "&opAttribute=" + _val + "&rand=" + randNum;
}
