When to use Binary counters

Optimost uses two types of counter, javascript counters and binary image counters. Javascript counters are javascript code placed on pages in a customer funnel to track activity as visitors travel through the customer's funnel. They usually pick up the Activity Tracking cookie that was dropped by Optimost on the test page, and send it back to the Optimost Server to indicate that visitor also made it to the page containing the counter.

Binary image counters have a bit more versatility. They are used in the following scenarios:

  1. In the customer funnel like a javascript counter, but when the customer does not allow third party code with javascript in their funnel, so we are forced to use third-party Activity Tracking and use binary image counters on the funnel pages.
  2. We are testing events on pages that are not redirects - say someone clicking to display a hidden DIV or clicking to see some section of a flash movie - we must use counters to track them, normally binary image counters.
  3. We are tracking a redirect, but do not want to use a clickthru (which uses a 302 redirect) for tracking.

This Wiki entry will give you an overview of what goes into Binary Counter, and then provide some code that can be used to cut and paste into a template if you want to use them for scenarios #2 and #3..

Triggering the Binary image counter

Here is the Optimost syntax for triggering a binary image counter (with the session identifier passed for activity tracking) is: 

<script>
var img= new Image();
var aN="action";
var aV="PageLoad"
img.src = "http://by.optimost.com/counter/0/-/0/event.gif?op000TestTestliid=" + escape("{{$creative.sessionId}}")+"&"+aN+"="+aV;
</script>

This img.src value consists of:

  1. The counter URL - http://by.optimost.com/counter/0/-/0/event.gif - which points to a counter created in the Optimost Agency console
  2. The Activity Tracking identifier name - op000TestTestliid in this example - which is associated with a subject in the Agency console. When you add a subject to the counter, it will appear in the image URL.
  3. The session identifier - {{$creative.sessionId}} in this example - which is an alphanumeric string associated with a page view. Its the value that goes with the Activity Tracking identifier.
  4. You may also add attrributes, here I passed "action" as a name and "PageLoad" as its value. More on that later.

This will result in a call to the Optimost servers, which returns a 1 x 1 pixel as the image. Since this is a javascript call, that pixel is iploaded but not displayed on the page.

Turning the code above into something useful

There are 2 major issues with the code above:

  1. If the Binary Image counter needs to be triggered on visitor "event" other than page load - say clicking on a link - the counter will need to be invoked by making a function call. If you put that code inside a function, when the function is done executing, the contents will be garbage collected by the browser and removed, and there is a chance the image has not made a call to the src URL yet before it is removed.
  2. If you make multiple counts/calls to the same image, unless the URL changes, it may be cached and thus all extra calls may not go to the Optimost Servers as the content it pulled from cache.

Here is the code as a function with these issues accounted for. 

<script>
var imgArray=[];
function setOffEvent(){
	var img= new Image();
	var rn=Math.floor(Math.random()*100000);
	img.src = "http://by.optimost.com/counter/0/-/0/event.gif?op000TestTestliid=" + escape("{{$creative.sessionId}}")+"&rn="+rn;
	imgArray.push(img);
}
</script>
<button onclick="setOffEvent();">Trigger Image counter via javascript</button>

Now the function allows us to defer its execution

Another way of adding a counter - adding a JS event to the DOM

Here, by adding it to the DOM, issue #1 goes away since there is nothing to garbage collect. And in this case, it's a JS call counter - the normal way we do it except it's not automatically pulling parameters as our counters do. And here we don't do it as an inline script tag using document.write, instead we add it to the DOM 

<script>
function setOffEvent(obj){
	var rn=Math.floor(Math.random()*100000);
	counterURL="http://by.optimost.com/counter/0/-/0/event.js?op000TestTestliid={{$creative.sessionId}}"+"&rn="+rn;
	var script = document.createElement('script');

	script.setAttribute('src',counterURL);
	script.setAttribute('type', 'text/JavaScript');
	document.documentElement.firstChild.appendChild(script);
}
</script>
<button onclick="setOffEvent();">Add counter to DOM, new spin on binary counter</button>

The image method is preferable, but if you have to do it this way, here's how.

Adding Binary Counters as Events

As shown in the button example, we can add binary counters as events - we want the counter to be triggered as the visitor performs some action/event, such as clicking on a button. There are two ways to attach that event, and lets look over how to do it each way.

Level 2 vs Level 0 issues: Passing Parameters into function and Bubbling of events

For level 2, there are some differences in how you will have to define the function that is called by the event, and also have to consider the difference between a Level 0 and Level 2 "event".

Function definition

When you do a Level 0 event, you pass the parameters in directly through the event, such as: 

<a id="testAnchor" href="demo.html" onclick="doMyOnclick(this, myValue,"span",true);">Look at Demo</a>

or if I add it to the anchor via jsvascript: 

<a id="testAnchor" href="demo.html">Look at Demo</a>
<script>
obj=document.getElementById("testAnchor");
if(obj){
	if(typeof(obj.onclick)!="function") obj.onclick=new Function('doMyOnclick(this, myValue,"span",true);return false;');
	else{
		temp=obj.onclick.toString();
		temp=temp.substring(temp.indexOf("{") + 1, temp.lastIndexOf("}"));
		obj.onclick=new Function('doMyOnclick(this, myValue,"span",true);return false;'+temp);
	}
}
</script>

For a level 2, you pass the "function" in a similar manner, but do not have to worry about previous existing events. So for a level 2 event, you would say: 

<a id="testAnchor" href="demo.html">Look at Demo</a>
<script>
obj=document.getElementById("testAnchor");
myValue=100;
if(obj){
	$opt.addEvent(obj,"click",new Function('doMyOnclick(myValue,"span",true);'),false);
}
</script>

For Level 2 functions, "this" does not exist in the traditional sense, please look at How to add Level 2 events for more information.

And, as a reminder, the addEvent function looks like this: 

<script>
$opt={};
$opt.addEvent=function(el,evt,func,bubble){
	if(!el || typeof(func) !="function"){return false;}
	bubble = bubble || false;
	var evtType=(window.attachEvent)?"attach":(window.addEventListener)? "add":"none";
	if(evtType == "attach"){el.attachEvent("on"+evt,func);}
	else if(evtType == "add"){el.addEventListener(evt,func,bubble);}
}	
</script>

Code for Level-2 Binary Counter tracking

I want these events added as Level-2 events, and have added tracking code and comments on creating that code. This section contains code and some explanations of issues you will encounter when creating calls to the events. The code sections will have red headings.

Code in the Setup block

Place this in the Setup block. If $opt exists, no need to create that object. 

<script>
$opt={};
$opt.addEvent=function(el,evt,func,bubble){
	if(!el || typeof(func) !="function"){return false;}
	bubble = bubble || false;
	var evtType=(window.attachEvent)?"attach":(window.addEventListener)? "add":"none";
	if(evtType == "attach"){el.attachEvent("on"+evt,func);}
	else if(evtType == "add"){el.addEventListener(evt,func,bubble);}
}
$opt.imgArray=[];
$opt.triggerCounter=function (aN,aV){
	var img= new Image();
	var rn=Math.floor(Math.random()*100000);
	img.src = "http://by.optimost.com/counter/0/-/0/event.gif?op000TestTestliid=" 
	+ escape("{{$creative.sessionId}}")+"&"+aN+"="+aV+"&rn="+rn;
	$opt.imgArray.push(img);
}	
</script>

In the Module of interest - how to define the event for a node

Here is how you would add the event to the nodes, as you find them. Here we will go over some issue having to do with closure and the different ways to define that function. You will see the recommended solutions at the bottom of the section.

Using new Function

As I said in #3, this is a simple example, link found by its "id". Your process of finding each link may be a bit more involved - it likely won't have an id - but this example should be enough to give you the idea of how it works. 

<a id="testAnchor" href="demo.html">Look at Demo</a>
<script>
obj=document.getElementById("testAnchor");
myValue=100;
if(obj){
	$opt.addEvent(obj,"click",new Function('$opt.triggerCounter("link","Demo");'),false);
}
</script>

BETTER way of adding event's function - not using new Function

The 3rd parameter passed into the $opt.addEvent function is the function to be called on the event. Using new Function to create that function is NOT recommended because because it is heavier, takes more overhead to create. And for static functions where the parameters don't change as the page runs, we can change this easily. So our example could more efficiently have been written: 

obj=document.getElementById("testAnchor");
myValue=100;
if(obj){
	$opt.addEvent(obj,"click",function(){$opt.triggerCounter("link","Demo")},false);
}

So for this simple example, use this.

creating a function for when you need to preserve loop values

Now lets say we are running a loop to find a bunch of anchors and are pulling some attribute out of each element that will be unique. Let's do a simple example where the index is passed in. For example: 

var as=document.getElementsByTagName("a");
for(var i=0,len=as.length;i<len;i++){
	$opt.addEvent(obj,"click",function(){$opt.triggerCounter("link",i)},false);
}

This will not work, since "i" is passed in as a variable, and when the click event actually happens, "i" will not have the value we intended to pass in,it will have the last value assigned. So our first thought is, ok, this is where you would use new Function, as in: 

var as=document.getElementsByTagName("a");
for(var i=0,len=as.length;i<len;i++){
	$opt.addEvent(obj,"click",new Function('$opt.triggerCounter("link","'+i+'");',false);
}

and this will work, but uses new Function. Is there a way around this.

Best Solution - using anonymous functions to pass in values in their own scope

What you want is for something to "lock" in the value as you go through the loop, and so you use an anonymous function to do this, so you have a function inside the function, and that inside function is defined when it is created, and the loop values are passed in and made static when the function is created. 

var as=document.getElementsByTagName("a");
for(var i=0,len=as.length;i<len;i++){
	$opt.addEvent(obj,"click",function (a,b){return function () {$opt.triggerCounter(a,b)}}("link",i),false);
}

Here "link" and the variable i are passed in, but each function in the lop has its own scope and keeps the correct value.

Code in the Module - Two recommended solutions

If you don't have any loops to worry about, use this: 

obj=document.getElementById("testAnchor");
myValue=100;
if(obj){
	$opt.addEvent(obj,"click",function(){$opt.triggerCounter("link","Demo")},false);
}

If you do have a loop to contend with, then do this: 

var as=document.getElementsByTagName("a");
for(var i=0,len=as.length;i<len;i++){
	$opt.addEvent(obj,"click",function (a,b){return function () {$opt.triggerCounter(a,b)}}("link",i),false);
}


AND DON'T FORGET

Keep a few things in mind:

  1. The image URL value used - what I show as http://by.optimost.com/counter/0/-/0/event.gif - will be determined by the Optimost Agency Console when you create the binary image counter in the Account's Setup area. Use that value.
  2. Also, remember the image needs to be secure on a secure page, the difference being by.optimost.com becomes by.essl.optimost.com/by on a secure page.
  3. You can pass attributes in - say you are using one binary image counter to track 4 buttons. You will need to pass in name-value pairs to the function, like say link=readme or link=callToAction, etc.
  4. If you are passing attributes, also remember to add them as attributes to the counter, either when you create it or shortly thereafter. In the Account of interest, when you go to the "Account Setup" section, select "Shared Counters" and access the counter of interest. Under "Set counter attributes", you need to add the "Name" and "Query string argument" - both the name - and the values will populate themselves.
  5. Make sure you add this counter to the subject of interest, or it will not track. When you access the counter of interest, click on "Change subjects" and select the appropriate ones.
  6. For the code in the "Module of Interest", you will have to find the element or elements you want to track through whatever means. The example below is just a simple link found based on its "id".
  7. Make sure you verify it works by testing it. You can use LiveHTTPheaders in Firefox, or IEliveHTTPheaders for IE. Look at the traffic and verify the activity tracking name value pair and any attributes are correct.
  8. AND create a report for the Staging wave to verify they are getting to our servers succesfully.
