Creating an Image counter - How and why do we do it?

Why would you need to do this? Well there are a number of reasons
·Want to count some state when a button is clicked, and view as counters are viewed in Wave reports - this is in fact the exact context in which this one was created.
·Can't track this click but want stats - client may not want us to do a 302 redirect for some reason, but still want info.
·Can't insert counters in flow - valid reason, of course you could use a click to track it, but whatever.

In the example below, they wanted to know which of the two radio buttons was selected when the visitor clicked on either the "click here to join" button or the "Try us free for 30 days" link.

You can take the code and view source, you will see that the function doOptInCheck() pulls the value of the selected radio button and sets a tag accordingly. The tag already contains the URL of the image counter, and the creative ID. We then add these attributes (host and testrun) to give us full reporting info.

Right now, we have are triggering an alert with the info. Notice that the {{$creative.sessionId}} is appearing as escaped content, in the template it would be given a session identifier. And for the real application, comment out the alert tags and uncomment the
img.src = tag;
statements.
