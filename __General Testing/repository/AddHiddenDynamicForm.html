Extreme DOM—Dynamic content, product details and a Form and its values on the fly from non-visible content
How and why do we do it?

For Dynamic content, in this case a the QVC's TSV (Today's Special Savings) page. The page is extremely dynamic, so there is a ton of dynamic code. I am pulling a bunch of stuff out of the DOM, saving it to javascript variables, then adding it to the page in the appropriate locations using document.writes.

The product details are pulled out in a number of ways - by ID and by TagName. Lines 256-282 in the code pull stuff out by ID, and notice that I make sure I have check for null values for the IDs (in which case the value will be an empty string).

The Form and its values are pulled out and re-rendered. This is the most confusing part of the code since I create a number of Custom Objects using a Constructor Function, which is just a regular javascript function, except that it just defines the structure of the object. This might seem like a lot of work, and it was when I defined the construtor functions, but means I just have to call it to create a new form variable value object, and I can just add it to the form object I create to start copying the form. So I define constructors for hidden inputs, text inputs, selects (pull-downs). I did not create one for Radio buttons, but I did that on the .NET form, so look at that for an example.

The Form areas, all in javascript, are encapsulated by comment blocks that look like this, look for them:

//****************** Form STUFF BELOW BELOW BELOW BELOW ****** blah blah blah *****************//
..
..
//****************** Form STUFF ABOVE ABOVE ABOVE ABOVE ****** blah blah blah *****************//
